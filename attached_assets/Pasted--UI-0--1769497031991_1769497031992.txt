目的：新歓イベント検索サイトを、モバイルファーストで“人間が作ったプロダクト感”が出るUIに改善し、ページを拡張する。
前提：認証なし / スマホメイン / 岡山ローカル限定。

# 0) グローバルナビ（最重要）
現状「イベント」「団体」しかないため、主要ページ（ナビ項目）を以下の5つに統一して追加・整理する：

1. イベント情報（Events）
2. 団体紹介（Groups）
3. 仲間を探す（Find Buddies）
4. 記事（Articles）
5. お問い合わせ（Contact）

※「HOME」は別枠で存在させ、ロゴ/ホームボタンでトップへ戻れるようにする。
※ただしナビの主要項目は上の5つを中心にする（スマホで見やすい構成）。

推奨：ヘッダーはシンプル、スマホはボトムナビ or ハンバーガーでも可。

# 1) HOME（トップページ）改善
- 「新歓ナビ」という表記はやめて、ナビ上は「HOME」として扱う。
- キャッチコピー例：
  「新しい出会いが、あなたの新しい可能性の扉を開く」
- ヒーロー直下に “安心の3要素” を短く提示（例）：
  1) 1人でも行きやすい新歓が見つかる
  2) 実際に行った人の声が見られる（レビュー）
  3) 岡山ローカルに絞って迷わない
- 主要CTAは1つに絞る：「イベント情報を見る」へ。

# 2) イベント情報（メイン機能）
- イベント検索/一覧：大学（3大学）/ 団体区分（部活・サークル）/ ジャンル / 1人参加しやすさ 等でフィルタ
- イベントカードを統一デザインにする：
  画像（16:9）→イベント名→団体名/大学→タグ（初心者歓迎/1人参加しやすさ/雰囲気）→日時/場所→CTA（詳細）
- 画像は現状プレースホルダーでOK（単色グラデ/抽象柄）。後でURL差し替え可能に。
- イベント詳細：
  - 詳細（日時/場所/説明/持ち物/参加の流れ）
  - レビュー一覧（投稿は後でも可。表示は先に用意）
  - 「主催団体を見る」導線（団体紹介へ）

# 3) 団体紹介（団体検索＋団体詳細）
- 団体検索/一覧：大学/団体区分/ジャンルで絞り込み
- 団体カードも画像（16:9）＋名前＋短い説明＋タグで統一
- 団体詳細：
  - 団体の雰囲気/活動内容/初心者向け情報
  - 主催イベント一覧（イベントと紐づけ）
  - 外部リンク（団体個別）：Instagram / Twitter(X) / 公式LINE
    - 登録されているものだけ表示
    - 外部リンクは新規タブ
    - 団体リンクとサイト公式リンクを混ぜない（場所を分ける）

# 4) 仲間を探す（新規追加）
目的：このイベントに行きたいが一緒に行く友達がいない人の背中を押す。
ただしMVPでは掲示板などは作らず、「公式LINEへ誘導するページ」として実装する。

ページ仕様：
- 見出し例：「1人が不安なら、仲間を探そう」
- 説明文：このページでは“仲間探し”の案内をする（個人情報注意）
- CTAボタン：『公式LINEで仲間を探す』
  - 現状、リンクURLは未設定でもOK（ダミー/プレースホルダー）
  - ただしUIとしてボタンと導線だけ作る
- 注意喚起：個人情報の扱い、無理に会わない等の安全ガイドを短く入れる

# 5) 記事（新規追加）
- 記事一覧（カード）：サムネ（プレースホルダー可）/ タイトル / 概要 / タグ / 日付
- 記事詳細ページ
- 記事カテゴリ（最低2系統のダミー記事を用意）：
  A) サークル・部活あるある系（新入生向けで軽い内容）
  B) 筆者の想い（なぜこのサイトを作ったのか、行動量を増やしたい理由）
- 記事はまず静的データ（JSON）でOK。後でDB化できる構造に。

# 6) お問い合わせ（新規追加）＋ FAQ（よくある質問）
お問い合わせページは2セクション構成にする：

A) お問い合わせ送信フォーム
- 種別：一般 / イベント掲載依頼
- 名前 or 団体名（任意）
- 大学（任意）
- 連絡先（メール or Instagram）
- 内容
- イベント掲載依頼の場合の追加項目（任意）：イベント名/日時/場所/説明/画像URL
- MVPは送信処理はダミーでOK（送信完了UI、保存は後回し可）

B) よくある質問（FAQ）
- アコーディオン（開閉）形式で見やすく
- ダミーFAQ例：
  - 掲載基準は？
  - どの大学が対象？
  - イベント掲載を依頼したい
  - 個人情報はどう扱う？
  - 団体のリンク（Instagram等）はどう追加する？

# 7) サイト公式リンク（グローバル）
サイト公式リンクとして、InstagramとNotionをヘッダーorフッターに常時表示する。
- サイト公式：Instagram / Notion（運営情報・更新ログ）
- 外部リンクは新規タブ
- 団体個別リンクとは視覚的に区別（位置・サイズ・説明）

# 8) デザイン改善方針（logic-driven UI）
- 余白でグルーピング（関連は近く、非関連は離す）
- 画面内のプライマリCTAは常に1つ
- カードUIを統一（画像→タイトル→メタ→タグ→CTA）
- アライメントは基本左揃えで統一し、揃えの種類を増やしすぎない
- モバイルファースト、最大幅、余白多め、角丸大きめ、影は薄め

# 9) 実装の進め方
- まず情報設計（ページ遷移）とデータ構造を提示してから実装に入ること
- MVPは静的データ（JSON）で実装してよい
- 後からDB/管理画面/承認フローに移行しやすい構造にする
